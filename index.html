Basic CSS Element Positioning

Position is one of the trickiest things to understand in web design, and we're largely going to be using a grid system to handle positioning our elements. Because of that, we're going to keep this exercise quick and dirty.

Additional Resource: CSS Positioning Explained By Building An Ice Cream Sundae

Basic Properties

  • position: static/relative/absolute/fixed
  • display: block/inline
  • width/height: value in x units
  • top/right/bottom/left: value in x units

Understanding CSS Units

  • %: percentage of parent container
  • px: simple measurement of pixels; 1px = 1 pixel
  • vw: viewport width; 1vw = 1% of the width of the viewport
  • vh: viewport height; 1vh = 1% of the height of the viewport
  • vmin: viewport minimum; 1vmin = 1% of the viewport's smaller dimension (width or height)
  • vmax: viewport maximum; 1vmax = 1% of the viewport's larger dimension (width or height)

Anatomy of a Layout

Here's a simple "sketch" of a page layout. It includes a pseudo header, navigation bar, sidebar (or aside), body with sections and articles, and footer.

We're going to recreate this layout together in the example layout page. Go ahead and open up the example-layout.html and example-layout.css files in Sublime, then we'll walk through this together.

Most of the elements are already on the page, but we will need to create the sections (and article) that appear in the box on the right.

We will need to add ids and classes to the elements that exist. We'll be using a combination of classes we create and classes from the TOAST grid system to style the page.

This grid system uses the class "grid__col" to determine that something is a column, then classes in the format "grid__col--1-of-6" to determine the column's width relative to its parent element. There are other classes, like "grid__col--centered" and "grid__col--push-3-of-6" that give us additional positioning options- we'll use those too!
